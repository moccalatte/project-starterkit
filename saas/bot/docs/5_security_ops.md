# 5. Security & Operations

## 5.1 Security
*   **Tokens**: Never commit API tokens. Use `.env`.
*   **Rate Limiting**: Prevent spam abuse.
*   **Input Sanitization**: Prevent injection if SQL is used.

## 5.2 Deployment
*   **Docker**: Dockerfile details.
*   **Service**: Systemd or Cloud Container.

## 5.3 Logging
*   Audit logs for all commands executed.
